选题目标
目标：共享学习资源，我们要做一个学习资源“共享”的网站，每一个资源都可以被下载，每一个人都可以上传自己想要上传的资源
具体内容：我们将共享学习资源分成四类：电子书、课件、笔记、试题。
（这也是我们在后期实现分类上传、和展示的依据）
选题依据
首先共享这个词很火，比如共享单车，共享知识的知乎等等，再加上我们周围的同学也有许多的需求，这就引起了我们做一个这样的网站的兴趣。
调研结果归纳：共享学习的网站或者应用有很多，如未名BBS， “大信科”公众号，盈利网站，积分机制的网站（如CSDN）,网盘等。但是和我们想做的东西还是有一定的区别。
总体设计思路
B/S结构（浏览器/服务器模式）
用户界面完全通过WWW浏览器实现，一部分事务逻辑在前端实现，但是主要事务逻辑在服务器端实现。浏览器通过Web Server 同数据库进行数据交互。
网页前端，服务器后端（前端页面显示，服务器后端用来从数据库存取所需要的资源信息）
从功能的角度出发：
用户：可以登陆，注册，编辑
资源：显示，上传，下载 ，搜索
显示：主页分两大类显示：电子书，学习资源【课件，笔记，试题】。还可以根据更细的分类查看网站上全部的资源。
	下载：在找到自己感兴趣的资源后，可以点击进入详情页，详情页有该资源的所有信息，包括下载按钮。点击可直接下载
	上传：有专门的上传页，登陆后可上传资源，按电子书，课件，笔记，试题等分类，由用户自己分类。
    搜索：头部有搜索栏，用户可以根据根据资源名进行搜索。

收获与体会
收获：
1、	对B/S结构（浏览器/服务器模式）有了更深的体会（用户界面完全通过WWW浏览器实现，一部分事务逻辑在前端实现，但是主要事务逻辑在服务器端实现。浏览器通过Web Server 同数据库进行数据交互。）
2、	一次完整的网站搭建，对网站功能的实现，后端环境的搭建，数据库的变淡设计，后端与前端如何实现交互有了更深的了解。
体会：
1、	写之前觉得需要完成的功能并不多，主要功能是网页的展示，资源的上传与下载功能，实现起来可能不会很难，但是真正要去实现的时候，发现有很多的细节需要处理。
2、	基本上实现了预期的功能，但是离应用还有一定的差距，等到以后有机会，会继续把相应的功能完善。
比如实现后台程序，让管理员管理更加方便。
实现上传和展示分开，用户可以上传任何资源，但是上传后的资源需要经过管理员审核和更好的编辑，再进行展示。
比如老师提到的版权问题，会想出一个很好的解决策略。
3、	 优化网站的UI设计和兼容性。
